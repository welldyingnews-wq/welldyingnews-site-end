## 협업 및 실행 지침 (입문자 가이드)

1. **최대한의 지원:** 나는 코딩 입문자이므로, 코드 작성, 로직 설계, 버그 수정 등 복잡한 작업은 AI인 네가 최대한 완성도 있게 처리해 줘.
2. **실행 명령의 분리:** 데이터베이스 마이그레이션(`npx prisma migrate`), 패키지 설치(`npm install`), 서버 실행 등 내가 터미널(Terminal)에 직접 입력해야 하는 '명령어'는 따로 명확히 구분해서 알려줘.
3. **토큰 및 리소스 관리:** 네가 직접 스크립트를 돌려보며 토큰을 과하게 쓰기보다는, 내가 실행해 볼 수 있는 '완성된 명령어'를 제공하는 방식으로 협업하자.
4. **쉬운 설명 포함:** 코드를 줄 때는 이 코드가 무엇을 하는지, 명령어를 입력하면 어떤 결과가 나오는지 입문자 눈높이에서 친절하게 설명해 줘.

## AI 기사 초안 작성 워크플로우 (cowork)

사용자가 "초안 작성 시작", "다음 기사", "기사 작성해줘" 등을 요청하면 아래 절차를 따른다.

### 사전 준비
- `scripts/collected/` 디렉토리에 스크래핑된 JSON 파일들이 있어야 함
- 없으면: `source .venv/bin/activate && python scripts/fetch_collect.py` 실행 안내

### 처리 절차 (1건당)

1. **미처리 파일 찾기**: `scripts/collected/`에서 JSON 파일 목록을 확인하고, AiDraft 테이블에 `original_url`이 이미 존재하는 건은 스킵한다.
2. **파일 읽기**: JSON에서 `scraped.text` (원문 본문)를 확인. `scraped.success`가 false이면 스킵.
3. **기사 초안 작성**: 아래 "기사 작성 규칙"에 따라 작성.
4. **DB 저장**: `save_draft.py`로 AiDraft에 저장.
5. **다음 건으로 자동 진행** (사용자가 "중지"할 때까지).

### 기사 작성 규칙

#### 절대 규칙
- **스크래핑된 원문(scraped.text)에 있는 사실만 사용한다.** 없는 내용을 추가하지 않는다.
- 추측, 해석, 개인 의견을 넣지 않는다.
- 번역체 표현을 사용하지 않는다 (예: `~라고 말했다` → `~라고 밝혔다`).
- `~것으로 보인다`, `~것으로 알려졌다` 등 추측성 표현 금지.
- `~에 대해서`, `~에 있어서` 같은 불필요한 조사구 최소화.
- 문장은 60자 이내로 작성한다.
- 직접 인용은 "..." 형태로 작성한다.

#### 기사 구성 (역피라미드)
- **제목**: 핵심 사실을 담은 한글 제목. 25자 이내 권장.
- **부제(subtitle)**: 리드문. 5W1H 핵심 사실 1~2문장.
- **본문(content)**: HTML 형식. `<p>` 태그로 문단 구분. 중요도순 역피라미드.
  - 해외 기사: 원문 출처를 본문 첫 문장에 자연스럽게 녹인다. (예: "AP통신에 따르면", "로이터 보도에 의하면")
  - 인용: 원문 인용 그대로 번역하여 "..." 형태로 사용.
  - 소제목이 필요하면 `<h3>` 태그 사용.
- **요약(summary)**: 200자 이내 기사 핵심 요약.
- **키워드(keywords)**: 쉼표로 구분된 3~5개 키워드.
- **출처(source_text)**: 원문 매체명 (예: "AP통신", "연합뉴스", "BBC").

#### 등급 판단
- **A1 (해설/분석)**: 심층적 주제, 다각적 분석 가능한 기사
- **A2 (스트레이트)**: 사실 전달 중심 기사 (대부분 이것)
- **B (단신)**: 단순 소식, 짧은 뉴스

